# VERIFY COMMAND

**Purpose:** Thoroughly investigate and verify that systems work correctly before making claims.

**What VERIFY does:**
1. **Code Investigation:** Actually read and analyze relevant code files
2. **Logic Verification:** Trace through actual code paths
3. **System Comparison:** Compare localStorage vs KV implementations
4. **Honest Assessment:** Only make claims after actual verification

**Usage:** `VERIFY [system/feature]` to thoroughly investigate before making any claims.
